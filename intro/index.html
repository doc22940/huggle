Hugo + gulp.js = Huggle.

Build a highly customizable personal blog blazing fast

Introduction

Hugo is a fast and flexible static site generator written in Go. It does an amazing job in organizing, templating and building contents.

There’s only one thing missing: a more organized way to write CSS/JavaScript your site will need. Of course we shouldn’t expect Hugo to do everything for us, while there are already kinds of solutions for this, which designers and front-end developers may found quite familar with:

  • Grunt - The most well-known JavaScript task runner
  • Brunch - An all-in-one toolset emphasized on building rich HTML5 apps
  • gulp.js - The streaming build system
  • Yeoman - Modern webapp scaffolding tool

There is alreay an existing static site building tool, called Assemble, which is built as a grunt plugin. It provides highly customizable configure options, and a seamless integration with existing Grunt plugins. Unfortunately, it’s not fast enough, and with all that big bunch of tasks to run, the gruntfile.js can be very hard to read and maintain.

It’s all about speed and ease, right? So here we are, trying to combine two useful tools, Hugo and gulp.js, which are both designed to be fast and easy to use.

By running two tasks respectively, generate html from organized markdowns and build assets from programmable css/js, we gain the maximum speed and flexibility.

What can you get?

  • Build a 200+ blogs site in seconds
  • Edit, save, and see live changes in less than on second
  • Organize contents in a meaningful way
  • Simple yet powerful golang html template, with partials and shortcodes support
  • Write site UI in coffee/stylus/sass/less, anything you like
  • Write browser code like your are in node.js

Quick start

  1. Install hugo // Long story short: download the binary
  2. Fork and clone this repo
  3. npm install
  4. Create a your-post.md under content/posts/ and edit it
  5. make server to see the result

If you want to edit css/js of your site:

  1. (optional) Install livereload
  2. make watch
  3. Edit files under /assets and /layouts

When you’d like to publish your site, see make gh-pages.

Also, don’t forget to look into docs on Hugo and gulp.js.

Trouble shooting

Rendering error: html/template: “xxx.html” is an incomplete template

It is because you have grammer error in your golange html template. Remember, you can only use double quote for strings.

{{ template "chrome/includes.html" . }}

Use " instead of ', and blankspace after the . are all mandatory.
